    Consommer un service Web en React

    • On consomme un service web au moyen d’un appel Ajax (asynchronous JavaScript and XML).
    • Il y a plusieurs bibliothèques de disponibles qu’on peut utiliser avec notre application React : Axios, Jquery Ajax et Fetch
    • Axios est simple d’utilisation et est très complet. Il dispose également d’une bonne couverture même sur les navigateurs plus anciens.

    Utilisation de Axios

    Pour l'installation de Axios, ouvrez un terminal et à la racine du projet dans lequel vous voulez l'utiliser entrez la commande suivante : 

    npm install axios

    Il faut ensuite importer axios dans les fichiers js où on veut l’utiliser.

    import axios from 'axios’;

    Voici un exemple de la structure d'un appel à un "endpoint" d'un api avec Axios

    axios.get('/maroute')
    .then((response) => {
    // Traitement si succès
    })
    .catch((error) => {
    // Traitement si erreur
    })
    .then(() => {
    // Traitement qui sera toujours exécuté
    })

    Quand faire l’Appel AJAX?

    • Quand on voudra obtenir nos informations pour les afficher dans une composantes. Habituellement on va le faire dans la méthode componentDidMount.
    • Une fois les données récupérées, on va utiliser setState pour mettre à jour la composante.

    Exemple de récupération de données au chargement du composant

    • On importe axios dans l’entête du fichier
    • Dans le constructeur, on crée un tableau vide qui va recevoir le résultat de l’appel.
    • On ajoute la méthode componentDidMount() qui va faire l’appel via Axios.
    • Quand l’appel réussi, on utilise setState() pour mettre à jour notre tableau et la composante.
    import React from 'react';
// On importe la bibliothèque Axios
import axios from "axios";

class ListeUtilisasteur extends React.Component {

    constructor(props) {
        super(props);
        this.state = {
            // On initialise ici un tableau vide qui va contenir notre résultat
            utilisateurs : [],
        }
    }

componentDidMount() {
    axios.get('https://jsonplaceholder.typicode.com/users')
        .then((response) => {
            const utilisateurs = response.data;
            // On récupère les données reçues et on modifie le tableau dans l'état
            this.setState({utilisateurs : utilisateurs})
        })
}

    On a vu que les appels à un api se font de manière asynchrone, on ne peut donc pas être assuré que les données seront disponible lors du premier rendu du composant. Une méthode pour se protéger de cette éventualité est de tester dans la fonction render si nos valeurs existe dans l'état local, ou bien ajouter une variable booléenne qui devient vrai au succés de l'appel à l'api.

    import React from 'react';
import axios from "axios";

class ListeUtilisasteur extends React.Component {

    constructor(props) {
        super(props);
        this.state = {
            // isLodad est faux tant que l'appel n'est pas terminé
            isLoaded: false,
            utilisateurs : [],
        }
    }

    componentDidMount() {
        axios.get('https://jsonplaceholder.typicode.com/users')
            .then((response) => {
                const utilisateurs = response.data;
                this.setState({
                    utilisateurs : utilisateurs,
                    isLoaded: true
                })
            })
    }

    render() {
        if(!this.state.idLoaded){
            return (
                // HTML à retourner tant que l'appel n'est pas terminé, un message 
                // de chargement par exemple
            )
        }

        return (
            // Le code "régulier"
        )
    }

    Syntaxe

    Il y a deux syntaxes possible pour l'utilisation de Axios, c'est selon votre préférence

    Méthode #1

    axios.post('urlDeLaRoute', {
    // données en paramêtre dans le body
    titre: 'Nouveau livre',
    isbn: '123456'
})
.then((resultat) => {
    // Traitement si l'appel a réussi
})

    Méthode #2

    axios({
    method: 'post',
    url: 'urlDeLaRoute',
    data: {
        titre: 'Nouveau livre',
        isbn: '123456'
    }
})
.then((resultat) => {
    // Traitement si l'appel a réussi
})

    Ajouter des informations dans l'entête de la requète

    Pour ajouter des informations dans l’entête de la requête, comme par exemple pour l’authentification, il suffit d’utiliser la clé headers.

    axios({
    method: 'post',
    url: 'urlDeLaRoute',
    data: {
        titre: 'Nouveau livre',
        isbn: '123456'
    },
    headers: {
        Authorization : 'Bearer monToken'
    }
})
.then((resultat) => {
    // Traitement si l'appel a réussi
})

    Créer une instance d’axios

    • Permet de ne spécifier par exemple qu’une fois l’url de base et d’ensuite n’ajouter que la route
    • Créez-vous un fichier Javascript qui contiendra le code de l’instance, ex. utils/Api.js
    // Api.js
import axios from "axios";

export default axios.create({
    baseURL: "https://swh2022-libapi.herokuapp.com",
    responseType: "json",
})
    • Ensuite on importe le fichier et on peut qu’ajouter la route lors de l’appel
    // La localisation du fichier est à ajuster
import api from '../utils/Api';

// ...

    componentDidMount() {
        // Ici on change axios pour api, le terme qu'on a défini dans le import
        api({
            method: 'post',
            // c'est ici qu'on peut seulement entrer la dernière partie de l'url qui
            // représente la route au lieu d'avoir à écrire 
            // https://swh2022-libapi.herokuapp.com/livres
            url: '/livres',
            data: {
                titre: 'Nouveau livre',
                isbn: '123456'
            }
        })
        .then((resultat) => {
            // Traitement si l'appel a réussi
        }) 
    }

// ...

    Votre code sera plus clair et lisible mais l'avantage majeur est que si l'api change de serveur, vous aurez à changer l'information qu'à un endroit dans le code.
