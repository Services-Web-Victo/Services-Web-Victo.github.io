    Gestions des formulaires HTML avec React

    Les formulaires html possède déjà leur propre « état » interne. Si on veut utiliser les données saisies dans nos composants React, on va créer ce qu’on appelle un « composant contrôlé ». Quand le champ du formulaire sera modifié, sur l’évènement onChange par exemple, on va mettre à jour une variable dans le state du composant qui va être « liée » au champ. Ensuite l’évènement submit va aussi être intercepté et traité dans le composant avec les données de l’objet state.

    Exemple

    J’ai un composant ListeLivres qui affiche un formulaire d’ajout de livre et un tableau avec tous les livres. Le formulaire est un composant ainsi que chaque ligne du tableau de livres (FormAjouterLivre, LigneLivre). Je veux pouvoir ajouter un livre par le formulaire et que la liste de livres se mettre à jour automatiquement.

    Dans le composant principal ListeLivre je vais avoir ma liste de livres dans un tableau nommé books défini dans l’objet state et aussi une fonction addBook() qui prendra en paramêtre un objet book qui contient les informations extrait du formulaire.

    La fonction addBook() va s’occuper de faire l’appel à l’api pour ajouter le livre et mettre à jour le tableau Books de l’objet state, ce qui va aussi mettre à jour l’affichage de la liste de livre.

    Dans le constructeur du composant, le tableau books est déclaré. On doit aussi « binder » les fonctions pour qu’elles soient reconnues, c’est ce qui est fait à la ligne this.addBook = this.addBook.bind(this);.

    // Extrait de la classe ListeLivre
constructor(props) {
    super(props);
    this.state = {
        books: []
    };

    this.addBook = this.addBook.bind(this);
}

    Dans la fonction addBook(), on fait l’appel à l’Api pour ajouter le livre et ensuite sur un résultat positif on ajoute le nouveau livre dans le tableau books de l’objet state. Dans le résultat, on commence par ajouter le id du nouveau livre dans l’objet book reçu en paramètre. Ensuite on crée un nouveau tableau avec le tableau books auquel on ajoute l’objet book. Finalement on met le tableau books de l’objet state à jour.

    // Extrait de la classe ListeLivre
addBook(book) {
    api({
        method: 'POST',
        url: '/books',
        data: {
            genreId: book.genreId,
            titre: book.titre,
            isbn: book.isbn
        }
    })
        .then((resultat) => {
            book.id = resultat.data.id;
            const books = [...this.state.books, book];
            this.setState({books: books});
    });
}

    Étant donné que la gestion de l’ajout se fait dans le composant ListeLivre et que la gestion du formulaire est dans FormAjouterLivre, quand on appelle FormAjouterLivre on doit lui passer en paramètre la fonction addBooks() pour pouvoir l’utiliser.

    <FormAjouterLivre addBook={this.addBook}/>

    Dans le composant FormAjouterLivre, je définis les variables dans le state qui contiendront les valeurs du formulaire. Je me suis aussi créé trois fonction :

    • handleChange, qui va modifier les variables du state quand un contrôle du formulaire sera modifié.
    • handleSubmit, qui va appelé la fonction addBook du composant ListeLivres reçu en props.
    • clearImputs, qui va simplement réinitialiser les variables state et en même temps les contrôles du formulaire.

    Ensuite dans le formulaire, je relis la fonction handleSubmit à l’événement onSubmit de la balise form et handleChange à l’événement onChange de chaque input. Les variables titre, genreId et isbn vont contenir les valeurs des inputs du même nom. Ensuite je bind les fonctions utilisées.

    // Constructeur de la classe FormAjouterLivre
constructor(props) {
    super(props);
    this.state = {
        titre: '',
        genreId: '',
        isbn: '',
    };

    this.handleChange = this.handleChange.bind(this);
    this.handleSubmit = this.handleSubmit.bind(this);
    this.clearInput = this.clearInput.bind(this);
}

    Dans la fonction handleChange, le code suivant permet de mettre à jour la variable de l’objet state qui a le même nom que le input du formulaire. Pour que ça fonctionne, il faut que la propriété name du input soit du même nom que la variable. On aurait aussi pu faire une fonction handleChange par input mais ça ajoute beaucoup de code pour rien. Ex. handleTitreChange qui met à jour this.state.titre.

    handleChange(event) {
    const name = event.target.name;
    const value = event.target.value;

    this.setState({
        [name]: value
    });
}

    Dans la fonction handleSubmit, je crée un objet book avec les valeurs de l’objet state et ensuite j’appelle la fonction addBook reçu en paramètre. J’empêche aussi le comportement par défaut du submit avec event.preventDefault(); 

    handleSubmit(event) {
    const book = {
        titre: this.state.titre,
        genreId: this.state.genreId,
        isbn: this.state.isbn,
    };
    this.props.addBook(book);

    event.preventDefault();
    this.clearInput();
}

    Finalement j’appelle la fonction clearInput qui s’occupe de réinitialiser les variables.

    clearInput() {
    this.setState({
        titre: '',
        genreId: '',
        isbn: '',
    });
}

    Voici une partie du formulaire 

    <form onSubmit={this.handleSubmit}>
    <table className='BooksTable'>
        <tbody>
            <tr>
                <td><label>Titre :</label></td>
                <td>
                    <input
                        type="text"
                        id="titre"
                        name="titre"
                        value={this.state.titre}
                        onChange={this.handleChange}
                        />
                </td>
            </tr>
